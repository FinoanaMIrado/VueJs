Apprendre
Vue.js

Vue.js est un framework JavaScript progressif pour construire des interfaces utilisateur réactives. Il est léger, flexible, et très facile à apprendre.

Pourquoi Vue.js ?

⚡

Réactif

Les données changent → l'interface se met à jour automatiquement.

🧩

Composants

Construis ton UI en blocs réutilisables et indépendants.

📦

Progressif

Ajoute Vue sur n'importe quelle page HTML, ou construis une SPA complète.

🛠️

Écosystème riche

Vue Router, Pinia, Vite — tout ce qu'il faut pour des apps modernes.

Installation rapide

La façon la plus simple : via un CDN dans ton HTML.

<!-- Dans votre HTML -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<!-- Avec Vite (recommandé pour projets) -->
npm create vue@latest mon-projet
cd mon-projet
npm install
npm run dev
💡 Vue 3 avec la Composition API et <script setup> est la syntaxe moderne recommandée.

L'Instance
Vue

Tout part d'un composant Vue. Avec la Composition API, on utilise <script setup> pour déclarer notre logique.

Structure d'un Composant

<!-- MonComposant.vue -->

<template>
  <div>
    <h1>{{ titre }}</h1>
    <p>Bienvenue, {{ nom }} !</p>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const titre = 'Mon App Vue'
const nom = ref('Alice')
</script>

<style scoped>
h1 { color: #41b883; }
</style>

Directives

Les
Directives

Les directives sont des attributs spéciaux préfixés par v-. Elles appliquent un comportement réactif au DOM.

v-bind — Lier des attributs

<!-- Forme longue -->
<img v-bind:src="imageUrl" v-bind:alt="description" />

<!-- Raccourci (: ) -->
<img :src="imageUrl" :alt="description" />
<button :disabled="estDesactive">Cliquer</button>

v-on — Écouter les événements

<!-- Forme longue -->
<button v-on:click="incrementer">+1</button>

<!-- Raccourci (@ ) -->
<button @click="incrementer">+1</button>
<input @input="onChange" @keyup.enter="valider" />

v-model — Liaison bidirectionnelle

<input v-model="texte" />
<p>Tu as tapé : {{ texte }}</p>

<!-- Équivaut à : -->
<input :value="texte" @input="texte = $event.target.value" />

v-if / v-else — Affichage conditionnel

<p v-if="score > 10">🎉 Gagné !</p>
<p v-else-if="score > 5">😐 Moyen</p>
<p v-else>😢 Perdu</p>

v-for — Listes

<ul>
  <li
    v-for="(item, index) in fruits"
    :key="item.id"
  >
    {{ index + 1 }}. {{ item.nom }}
  </li>
</ul>
⚠️ Toujours utiliser :key unique avec v-for pour la performance.

Réactivité

Réactivité &
Composition API

ref() — Données réactives simples

import { ref } from 'vue'

const compteur = ref(0)
const nom = ref('Vue')

// Dans le script, on accède avec .value
compteur.value++
console.log(nom.value) // 'Vue'

// Dans le template, Vue retire .value automatiquement
// <p>{{ compteur }}</p> ← pas de .value ici

reactive() — Objets réactifs

import { reactive } from 'vue'

const utilisateur = reactive({
  nom: 'Alice',
  age: 25,
  admin: false
})

utilisateur.age++ // Réactif directement, pas de .value

computed() — Propriétés calculées

import { ref, computed } from 'vue'

const prix = ref(100)
const tva = ref(0.2)

const prixTTC = computed(() => {
  return prix.value * (1 + tva.value)
})

// Recalculé automatiquement si prix ou tva change
// <p>{{ prixTTC }}</p> → 120

watch() — Surveiller des changements

import { ref, watch } from 'vue'

const recherche = ref('')

watch(recherche, (newVal, oldVal) => {
  console.log(`Changé: ${oldVal} → ${newVal}`)
  // Lancer une recherche API ici
})

Composants

Les
Composants

Les composants sont les briques de base d'une app Vue. On les imbrique pour construire des interfaces complexes.

Props — Passer des données au composant enfant

<!-- Enfant : BoutonAction.vue -->
<template>
  <button :class="type">{{ label }}</button>
</template>

<script setup>
defineProps({
  label: { type: String, required: true },
  type:  { type: String, default: 'primary' }
})
</script>

<!-- Parent : utilisation -->
<BoutonAction label="Valider" type="success" />

Emits — Communiquer avec le parent

<!-- Enfant -->
<script setup>
const emit = defineEmits(['valide', 'annule'])

function soumettre() {
  emit('valide', { data: 'résultat' })
}
</script>

<!-- Parent -->
<MonFormulaire @valide="handleValide" />

Slots — Contenu flexible

<!-- Carte.vue -->
<template>
  <div class="carte">
    <h3>{{ titre }}</h3>
    <slot />  <!-- contenu injecté ici -->
  </div>
</template>

<!-- Utilisation -->
<Carte titre="Info">
  <p>Ce texte va dans le slot !</p>
</Carte>

Lifecycle Hooks

import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('Composant monté dans le DOM')
  // Appels API, event listeners...
})

onUnmounted(() => {
  console.log('Composant détruit')
  // Nettoyage des listeners
})
Avancé

Les
Composables

Les composables sont des fonctions qui encapsulent et réutilisent de la logique avec état. C'est l'équivalent Vue des hooks React.

Exemple : useCompteur

// composables/useCompteur.js
import { ref, computed } from 'vue'

export function useCompteur(initial = 0) {
  const valeur = ref(initial)

  function incrementer() { valeur.value++ }
  function decrementer() { valeur.value-- }
  function reset() { valeur.value = initial }

  const estPositif = computed(() => valeur.value > 0)

  return { valeur, incrementer, decrementer, reset, estPositif }
}

// Dans un composant
<script setup>
import { useCompteur } from './composables/useCompteur'
const { valeur, incrementer, reset } = useCompteur(10)
</script>

Exemple : useFetch

// composables/useFetch.js
import { ref } from 'vue'

export function useFetch(url) {
  const data    = ref(null)
  const loading = ref(true)
  const error   = ref(null)

  fetch(url)
    .then(r => r.json())
    .then(d => { data.value = d; loading.value = false })
    .catch(e => { error.value = e; loading.value = false })

  return { data, loading, error }
}
💡 Convention : toujours préfixer les composables par use (useTodo, useAuth, useLocalStorage...)

Todo App
Interactive

Appliquons tout ce qu'on a appris ! Cette Todo App utilise : ref, computed, v-model, v-for, v-if, @click.

Todo App — Vue.js en action

    Le Code Source

    <template>
  <div>
    <input v-model="nouvelleTache" @keyup.enter="ajouter" />
    <button @click="ajouter">Ajouter</button>

    <ul>
      <li
        v-for="todo in tachesFiltrees"
        :key="todo.id"
        :class="{ done: todo.fait }"
      >
        <input type="checkbox" v-model="todo.fait" />
        {{ todo.texte }}
        <button @click="supprimer(todo.id)">✕</button>
      </li>
    </ul>

    <p>{{ tachesRestantes }} tâches restantes</p>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

const nouvelleTache = ref('')
const filtre       = ref('all')
const taches       = ref([
  { id: 1, texte: 'Apprendre Vue.js', fait: false },
  { id: 2, texte: 'Faire un projet',  fait: false }
])

const tachesFiltrees = computed(() => {
  if (filtre.value === 'active') return taches.value.filter(t => !t.fait)
  if (filtre.value === 'done')   return taches.value.filter(t =>  t.fait)
  return taches.value
})

const tachesRestantes = computed(() =>
  taches.value.filter(t => !t.fait).length
)

function ajouter() {
  if (!nouvelleTache.value.trim()) return
  taches.value.push({
    id: Date.now(),
    texte: nouvelleTache.value,
    fait: false
  })
  nouvelleTache.value = ''
}

function supprimer(id) {
  taches.value = taches.value.filter(t => t.id !== id)
}
</script>
